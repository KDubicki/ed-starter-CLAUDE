---
name: review
description: Use this skill when the user asks for a code review, audit, or quality assessment of TypeScript/JavaScript code — either pasted directly or specified by a file path. Triggers include "code review", "check this code", "review the code", "do a review", "code audit", "review this PR", as well as any question about finding bugs or assessing quality.
------

Conduct a detailed code review for the following code or the specified file.

Check for:
1. Business logic correctness
2. Error handling and edge cases
3. Security - particularly input data validation
4. Readability and naming conventions
5. Compliance with TypeScript strict mode

For each issue, provide: severity level (blocker / warning / suggestion), a description of the issue, and a fix suggestion.

If the code is good - say so explicitly and point out what was done well.
